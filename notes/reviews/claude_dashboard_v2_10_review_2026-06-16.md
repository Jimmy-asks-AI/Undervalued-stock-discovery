Jimmy,我是外部审查员。以下是对当前版本的审查意见:

## 一、做得好的地方

### 量化研究部分
1. **状态管理严格**: research_only 状态贯穿所有输出,即使内部测试有收益也不夸大为信号,避免误导使用
2. **边界诚实**: 承认"压力门控和小范围参数扰动没有把低估超跌转化为稳定行业选择 alpha",不假装有 alpha
3. **参数空间完整**: 50 组参数×3 持有期=150 行完整记录,覆盖条件观察、样本不足、审计失败三类状态
4. **数据结构清晰**: parameter_nav_samples 13850 行,parameter_event_returns 810 行,支持多维度分析
5. **版本隔离明确**: 每个版本独立目录 outputs/industry_realtime_pressure_sensitivity_v2_10,不混用历史数据

### Dashboard 部分
1. **数据到 UI 映射透明**: scripts/build_dashboard_dataset.py 读取 outputs 生成 dashboard_data.json,流程清晰
2. **参数实验台功能完整**: 支持 4 个维度(压力门槛/动量陷阱/权重扰动/现金假设)调整,联动显示样本/收益/净值
3. **视觉化合理**: ECharts 处理时间序列和散点图,lucide-react 图标一致,不引入多余样式库
4. **README 覆盖使用方式**: 明确说明如何启动、如何更新数据、如何理解参数实验台
5. **根目录整洁**: 只保留 README.md,不污染工作区

## 二、主要问题(按严重程度排序)

### 严重问题

**P1: 参数实验台没有样本量保护**
- 调参时可以快速把样本压到极低(例如 pressure_thresh=0.99 后只剩 5 个样本),但 UI 没有显式警告
- 用户可能误认为"高收益+低样本"是有效信号,实则是过拟合
- dashboard_data.json 中 parameter_summary 已有 sample_count,但前端未做门槛式提示(如"样本<30 则禁用或标记")

**P2: 最佳参数的相对收益为负**
- 60 日平均 -0.383%,持仓相对净值 0.997,即使"最佳"也未跑赢基准
- 但 dashboard 总览页未用红字/图标明确提示"所有参数组合均未获得正相对收益"
- 用户可能误读"最佳参数=有效策略"

**P3: V2.10 结论未在 dashboard 首页展示**
- 根目录 README 有结论,但 dashboard 未在"总览"页加醒目提示
- 用户需深入"回测表现"+"参数敏感性"才能发现"无参数通过候选门槛"
- 结论应在 dashboard 第一屏可见,用 banner 或卡片突出

### 中等问题

**P4: 参数实验台缺少历史锁定功能**
- 当前只展示 V2.10 冻结网格,但用户无法保存/对比"我刚才的参数 vs 默认参数"
- 如果用户多次调参,容易忘记起点,难以及时止损

**P5: 样本外与非重叠收益差异大**
- 最佳参数样本外 +0.393%,非重叠 -0.118%,方向相反
- dashboard 未解释这两者的定义和差异来源,用户可能困惑

### 轻微问题

**P6: Warnings=0 但未展示审计规则**
- dashboard_data.json warnings=0,但没有展示审计规则列表
- 用户无法确认"审计通过"究竟检查了什么

**P7: 参数实验台的现金假设未说明单位**
- 如果用户输入 0.5,是 50% 现金还是 0.5% 现金?
- UI 缺少 tooltip 或单位标签

## 三、后续迭代方向(按优先级排序)

### 应立即做
1. **在参数实验台增加样本量硬限制**: 当样本<30 时,禁用调整或显示红色警告"样本不足,可能过拟合"
2. **在 dashboard 首页增加 V2.10 结论 banner**: 用黄底黑字提示"当前版本无参数组合通过候选门槛,所有参数相对收益为负"

### 值得做
3. **增加样本量与相对收益的散点图**: 在"参数敏感性"页,横轴样本量,纵轴相对收益,直观展示"高收益=低样本"模式
4. **增加参数历史对比**: 在参数实验台增加"保存/加载配置"按钮,允许用户保存 3 个快照进行对比

### 不应做
5. **不要增加"自动推荐参数"功能**: 当前未证明有效,自动推荐会诱导过度信任;应保持"用户自己调参但 UI 明确警告"

## 四、如果只做一个最小但高价值的迭代

**建议:在参数实验台增加样本量硬限制 + 红色警告**

### 为什么
1. **风险控制**: 防止用户误把"高收益+低样本"当作有效策略,这是当前最大的误用风险
2. **成本低**: 只需在前端加一个 if 判断,无需重新跑数据或改后端
3. **符合研究诚实原则**: 既然数据已显示"无参数通过候选门槛",UI 就应阻止用户误读

### 具体做法
- 在参数实验台顶部增加一条警告:"当前样本数低于 30 时,结果可能过拟合,请谨慎参考"
- 当样本<30 时,将样本数标红,并在"相对收益"旁加警告图标
- 在 dashboard_data.json 中增加"min_safe_sample: 30"字段,前端统一读取

## 五、对"参数实验台"的风险提醒

### 风险 1: 调参诱导过拟合
- **风险等级**: 高
- **场景**: 用户多次调整参数,直到看到"高相对收益",误以为找到有效策略,实则是过拟合
- **控制措施**:
  - UI 上增加"参数实验台仅供理解参数敏感性,不建议用于实盘策略"提示
  - 当样本<30 时,禁止"保存配置"或"导出结果"功能
  - 每次调整参数后,显示"当前参数组合未通过 V2.10 候选门槛"提示

### 风险 2: 数据依赖单一版本
- **风险等级**: 中
- **场景**: 用户只看到 V2.10 结果,误以为参数空间只有 50 组,忽略历史版本的失败
- **控制措施**:
  - 在参数实验台增加"版本说明":V2.10 覆盖 50 组参数,历史版本已测试 XXX 组参数
  - 在"方法说明"页增加"参数迭代历史",展示 V2.0-V2.10 的演化

### 风险 3: 样本外与非重叠收益误导
- **风险等级**: 中
- **场景**: 用户看到样本外 +0.393%,误以为有效,忽略非重叠 -0.118%
- **控制措施**:
  - 在参数实验台同时显示三个指标:60日、样本外、非重叠
  - 在"方法说明"页增加"样本外 vs 非重叠"定义解释
  - 当三个指标方向不一致时,显示警告"收益不稳定,需谨慎参考"

---

以上为审查意见。当前版本诚实度高,但参数实验台的 UI 未充分提示风险,建议优先加样本量保护。
